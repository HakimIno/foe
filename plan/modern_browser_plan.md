# 🚀 แผนการพัฒนา Modern Browser (แรงบันดาลใจจาก Zen Browser)

เอกสารนี้คือแผนการทำงาน (Roadmap) และ Checklist เพื่ออัปเกรดเบราว์เซอร์ของเรา (Rust + Slint + Wry) ให้มีหน้าตาทันสมัย สวยงาม และฟีเจอร์ล้ำหน้าเทียบเท่า **Zen Browser** โดยจะแบ่งการพัฒนาออกเป็นเฟส (Phases) เพื่อให้ง่ายต่อการติดตามและตรวจสอบ

---

## 🟢 Phase 1: ปรับโฉม UI สู่ Vertical Tabs (เริ่มต้นทันที)
*เป้าหมาย: เปลี่ยนโครงสร้างแอปพื้นฐานให้เป็นแถบด้านข้าง (Sidebar) แบบยุคใหม่*

- [ ] **1. ปรับแก้ Slint UI Layout**
  - เปลี่ยนแถบ Tab (แนวนอนด้านบน) ให้กลายเป็น Sidebar (แนวตั้งด้านซ้าย)
  - นำ Address Bar และเครื่องมือควบคุม (Back, Forward, Reload) ย้ายลงไปไว้ใน Sidebar หรือทำเป็น Floating Bar
- [ ] **2. ปรับพิกัดของ Wry WebView**
  - แก้ไขพิกัดการเกิดของ WebView (`wry_engine`) จากที่เว้นพื้นที่ด้านบน (Y=76) ไปเป็นเว้นพื้นที่ด้านซ้าย (X=ความกว้างของ Sidebar)
- [ ] **3. ทำระบบพับเก็บ Sidebar (Collapse/Expand)**
  - ผู้ใช้สามารถกดซ่อน/แสดงแถบด้านข้างได้ เพื่อขยายพื้นที่อ่านเว็บให้เต็มจอ

---

## 🟡 Phase 2: ระบบจัดการ Workspaces (จัดกลุ่มแท็บ)
*เป้าหมาย: เพิ่มฟีเจอร์จัดระเบียบแท็บสำหรับผู้ที่มีแท็บเปิดเยอะๆ*

- [ ] **1. โครงสร้างข้อมูลฝั่ง Rust**
  - เพิ่มตัวแปร Workspace/Group เข้าไปในโครงสร้างของ Tab
  - สร้างระบบสลับ Workspace (Active Workspace)
- [ ] **2. อัปเดต Slint UI**
  - เพิ่มไอคอนหรือแถบเล็กๆ (Sidebar ริมสุด) สำหรับกดสลับ Workspace (เช่น งาน, ส่วนตัว, โซเชียล)
- [ ] **3. การจัดการ WebView**
  - แก้ไข `WryEngine` ให้ซ่อน (`set_visible(false)`) WebViews ทั้งหมดที่ไม่ได้อยู่ใน Workspace ปัจจุบัน

---

## 🟠 Phase 3: ระบบแบ่งหน้าจอ (Split Views)
*เป้าหมาย: เปิด 2 เว็บในหน้าต่างเดียวกันแบบ Side-by-side*

- [ ] **1. อัปเดตตรรกะ Wry Engine**
  - รองรับการแสดงผล WebView 2 ตัว (หรือมากกว่า) พร้อมกัน
  - คำนวณ Bounds (พิกัด) แบ่งครึ่งหน้าจออัตโนมัติ (เช่น จอซ้าย กว้าง 50%, จอขวา กว้าง 50%)
- [ ] **2. ปุ่มควบคุมฝั่ง UI**
  - เพิ่มปุ่ม "Split" ใน Tab เพื่อแบ่งหน้าต่างปัจจุบันคู่กับ Tab อื่น

---

## 🔵 Phase 4: Command Palette (แผงควบคุมอัจฉริยะ)
*เป้าหมาย: ควบคุมทุกอย่างด้วยคีย์บอร์ด โดยไม่ต้องจับเมาส์*

- [ ] **1. สร้าง Floating UI**
  - ออกแบบกล่องค้นหาแบบลอยตรงกลางจอ (สไตล์ Spotlight หรือ Alfred)
- [ ] **2. ผูกคีย์ลัด (Hotkeys)**
  - ดักจับปุ่ม `Cmd+K` หรือ `Ctrl+K` เพื่อเรียกหน้าต่างนี้ขึ้นมา
- [ ] **3. ระบบค้นหาแบบรวมศูนย์ (Omnibox)**
  - เมื่อพิมพ์ค้นหา จะแสดงผลลัพธ์ทั้งจาก: เปิดแท็บเก่า, ประวัติการเข้าชม (History), หรือค้นหาใน Google ทันที

---

## 🟣 Phase 5: รีดประสิทธิภาพและความสวยงามขั้นสุด
*เป้าหมาย: ยกระดับความพรีเมียมให้ผู้ใช้ร้อง "ว้าว" เมื่อใช้งาน*

- [ ] **1. ความสวยงาม (Aesthetics)**
  - ใช้ Effect กระจกขุ่น (Glassmorphism / Blur) บน Sidebar
  - แอนิเมชันลื่นไหลเวลาสลับแท็บ หรือย่อขยาย Sidebar
- [ ] **2. ลดคอขวด IPC (Performance)**
  - อัปเดตการดึงรูปภาพ ประวัติ หรือ Bookmark ให้ทำงานบน **Custom Protocol (`app://`)** ตามกฎเหล็กที่เราคุยกัน
  - เปลี่ยนวิธีส่งข้อมูลแท็บไปยัง Slint UI ให้ส่งเฉพาะแท็บที่เปลี่ยน (Delta Update) แทนการส่งข้อมูลใหม่หมดทั้งก้อน
